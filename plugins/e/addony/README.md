# Аддоны

## 🧩 Аддоны TreexClans

Добро пожаловать в раздел **«Аддоны»** — это страница общего ознакомления с модульной системой TreexClans. Здесь объясняется, что такое аддоны, где они находятся, как они работают и какие существуют.

> 📘 Подробное руководство по созданию и подключению аддонов находится на отдельной странице — Разработка аддонов.

***

### 📖 Что такое аддон

**Аддон TreexClans** — это дополнительный модуль, расширяющий функционал клановой системы без изменения ядра плагина.\
Каждый аддон представляет собой самостоятельный `.jar`‑файл, который можно легко установить, удалить или обновить.

**С помощью аддонов можно:**

* добавлять новые возможности для кланов;
* внедрять визуальные или экономические механики;
* интегрировать кланы с другими системами сервера;
* создавать собственные квесты, рейтинги, события и интерфейсы.

***

### 📦 Расположение аддонов

Все аддоны находятся в папке:

```
/plugins/TreexClans/addons/
```

Пример структуры:

```
plugins/
 └── TreexClans/
      └── addons/
           ├── ClanShop.jar
           ├── ClanTop.jar
           └── ExampleAddon.jar
```

**Основные правила:**

* 1 `.jar`‑файл = 1 аддон;
* обновление выполняется заменой файла;
* при запуске TreexClans автоматически находит и активирует все аддоны.

***

### ⚙️ Как работает система аддонов

TreexClans при запуске:

1. Сканирует папку `addons/`;
2. Проверяет каждый файл на совместимость;
3. Загружает аддоны в порядке инициализации;
4. Активирует их, интегрируя с ядром системы.

Таким образом, все модули работают независимо, но через единый API взаимодействуют с TreexClans.

***

### 🪶 Преимущества системы

* **Модульность** — каждая функция оформлена отдельным пакетом;
* **Гибкость** — можно включать или отключать любые модули;
* **Безопасность** — ядро плагина остаётся неизменным;
* **Совместимость** — аддоны не мешают друг другу;
* **Масштабируемость** — система легко расширяется.

***

### 💾 Файлы и конфигурация

Каждый аддон имеет свою директорию для хранения файлов:

```
/plugins/TreexClans/addons/<id_аддона>/
```

В ней могут располагаться:

* `config.yml` — основной конфигурационный файл;
* дополнительные `.yml` файлы (например, настройки, данные, квесты);
* кэш, временные и вспомогательные файлы.

Эта структура создаётся автоматически при первом запуске аддона.

***

### ⚖️ Совместимость и зависимости

Некоторые аддоны могут зависеть от других. Если необходимый модуль отсутствует или несовместим, TreexClans уведомит об этом в консоли и не запустит зависимые аддоны.

> 🔍 Система следит за порядком загрузки, чтобы все модули запускались корректно и без конфликтов.

***

### 💡 Примеры встроенных и популярных аддонов

| 🔧 Название    | 📝 Описание                                   |
| -------------- | --------------------------------------------- |
| **ClanShop**   | Клановый магазин предметов, валют и апгрейдов |
| **ClanQuests** | Система заданий и достижений клана            |
| **ClanEvents** | Ивенты, сражения и PvP‑механики               |
| **ClanTop**    | Альтернативные рейтинги и лидерборды          |
| **ClanBank**   | Клановый банк и система переводов             |
| **ClanDecor**  | Настройка внешнего вида и символики клана     |

> 🧩 Этот список — лишь примеры. Аддоны могут быть как встроенными, так и разработанными сторонними авторами.

***

### 🚀 Зачем нужны аддоны

Аддоны делают TreexClans полноценной экосистемой, где каждая часть системы может быть модулем:

* администраторы гибко управляют функционалом;
* разработчики добавляют свои идеи без модификации ядра;
* серверы адаптируют систему под собственный стиль игры (RPG, Grief, SMP и т.д.).
